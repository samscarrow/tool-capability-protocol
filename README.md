## üèóÔ∏è TCP Security Architecture

### First-Order Encoding (24 bytes)
```
TCP Descriptor v2 (24 bytes total)
‚îú‚îÄ‚îÄ Magic + Version (6 bytes)     # TCP\x02 + version info
‚îú‚îÄ‚îÄ Command Hash (4 bytes)        # Unique command identifier  
‚îú‚îÄ‚îÄ Security Flags (4 bytes)      # Risk level + capability flags
‚îú‚îÄ‚îÄ Performance Data (6 bytes)    # Execution time + memory + output size
‚îú‚îÄ‚îÄ Reserved Fields (2 bytes)     # Command length + future use
‚îî‚îÄ‚îÄ CRC16 Checksum (2 bytes)      # Integrity verification
```

### Second-Order Hierarchical Encoding
```
Tool Family Encoding (Multi-command tools like git, docker, kubectl)
‚îú‚îÄ‚îÄ Parent Descriptor (16 bytes)
‚îÇ   ‚îú‚îÄ‚îÄ Magic TCP\x03 (4 bytes)         # Hierarchical version
‚îÇ   ‚îú‚îÄ‚îÄ Family Hash (4 bytes)           # Tool family identifier
‚îÇ   ‚îú‚îÄ‚îÄ Common Properties (2 bytes)     # Shared characteristics
‚îÇ   ‚îú‚îÄ‚îÄ Command Count + Risk Floor (2 bytes)
‚îÇ   ‚îú‚îÄ‚îÄ Family Metadata (2 bytes)       # Tool type classification
‚îÇ   ‚îî‚îÄ‚îÄ CRC16 (2 bytes)
‚îî‚îÄ‚îÄ Delta Descriptors (6-8 bytes each)
    ‚îú‚îÄ‚îÄ Subcommand Hash (1 byte)        # Command identifier
    ‚îú‚îÄ‚îÄ Risk Delta (1 byte)             # Risk above family floor
    ‚îú‚îÄ‚îÄ Specific Capabilities (2 bytes)  # Unique command flags
    ‚îú‚îÄ‚îÄ Performance Profile (1 byte)     # Log-encoded time/memory
    ‚îî‚îÄ‚îÄ Metadata (1-3 bytes)            # Command properties
```

## üìã TCP Security Classification System

### Risk Levels (Proven in Practice)
- **CRITICAL (üíÄ)**: Data destruction possible (rm, dd, mkfs, bcachefs format)
- **HIGH_RISK (üî¥)**: System modification (chmod, mount, git rebase, bcachefs fsck)  
- **MEDIUM_RISK (üü†)**: File operations (cp, mv, git commit, bcachefs device add)
- **LOW_RISK (üü°)**: Information gathering (ps, find, git log, bcachefs list)
- **SAFE (üü¢)**: Read-only operations (cat, echo, git status, bcachefs show-super)

### Security Flags (Bit-encoded)
```
Bit  Flag                    Description
0    SAFE                   Read-only, no side effects
1    LOW_RISK               Information gathering  
2    MEDIUM_RISK            File/data modification
3    HIGH_RISK              System state changes
4    CRITICAL               Potential data destruction
5    (reserved)             Future use
6    REQUIRES_ROOT          Needs sudo/root privileges  
7    DESTRUCTIVE            Can permanently delete data
8    NETWORK_ACCESS         Makes network connections
9    FILE_MODIFICATION      Modifies file contents
10   SYSTEM_MODIFICATION    Changes system state
11   PRIVILEGE_ESCALATION   Can escalate privileges
12-15 (reserved)           Future security flags
```
